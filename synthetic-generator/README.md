# Synthetic RDF 1.2 generator

Deterministic generator of synthetic RDF 1.2 Turtle documents used by the
conversion and Neo4j-loading experiments in this repository.

For a fixed `--profile`, `--count`, and `--seed`, the generated bytes are
stable. In the evaluation, `--count` is the **scale factor**: scale factors 200,
400, 600, 800, and 1,000 produce the five checked-in synthetic datasets.

## Requirements and installation

- Python 3.10 or newer.
- No third-party runtime dependencies.

Run directly from a checkout:

```bash
./synthetic_gen --profile all --count 2 --seed 7 -o dataset.ttl
```

Install the CLI locally:

```bash
python3 -m pip install -e .
rdf12-synthetic --profile all --count 2 --seed 7 -o dataset.ttl
```

## Options

List available profiles:

```bash
rdf12-synthetic --list-profiles
```

Write a single profile to standard output:

```bash
rdf12-synthetic --profile rdf-reifies --count 5
```

Add the optional RDF 1.2 version directive:

```bash
rdf12-synthetic --version-directive --profile all --count 1
```

Available profiles are:

- `basic`
- `predicate-object-list`
- `literals`
- `blank-nodes`
- `collections`
- `rdf-reifies`
- `asserted-and-reified`
- `triple-term-object`
- `nested-triple-terms`
- `reifier-label`
- `multiple-reifies`
- `one-reifier-many-propositions`
- `annotation-block`
- `irirefs`
- `mixed-rdf12`

The `all` profile emits every listed feature family in a fixed order.

## Evaluation datasets

The synthetic experiment uses:

```bash
for scale_factor in 200 400 600 800 1000; do
  rdf12-synthetic --profile all --count "$scale_factor" --seed 7 \
    -o "/tmp/synthetic-${scale_factor}.ttl"
done
```

`experiments/run_experiments.py` performs this generation directly through the
Python API and records the actual byte, line, and parsed-triple counts.
